Overview

Built to learn the full Webflow toolkit — by building something that looks and feels production-ready.

Primer is a fictional fintech product — a digital payments and finance app used as the vehicle for mastering real-world Webflow techniques
Full SaaS landing page architecture — hero, social proof, feature sections, pricing toggle, SDK grid, and word marquees
Demonstrates command of Webflow Interactions — scroll reveals, toggle states, animated marquees, hover effects, and responsive breakpoints

Brief

Build something that converts

The course brief was open-ended: design and build a landing page that feels real. Not a template exercise — a page a real product team would ship. The constraint was the tool: Webflow only, no custom code.

Approach

Structure first, interactions second

Chose fintech as the domain — high visual standards, clear user intent, recognisable patterns. Designed the page architecture before touching Webflow: hero → proof → features → pricing → CTA. Then built each section with intentional interaction design.

Outcome

A page that earns its complexity

Every interaction on the page serves the conversion goal. The pricing toggle, animated marquees, feature card grid, and scroll-triggered reveals are not decorative — they reflect how real SaaS products communicate value across a long-form landing page.

Design

Minimal input, maximum clarity

The visual direction follows a proven SaaS pattern — light background, strong typography hierarchy, mobile app mockups as the primary visual evidence. The design stays clean so the product does the talking.

Floating mobile UI screenshots in the hero create immediate product credibility. Every section below narrows from broad value proposition to specific feature to pricing — a deliberate funnel structure.

Feature section — mobile mockup with layered card composition and social proof

Key Feature

Interactions that earn attention

Every animated element serves the layout — not the other way around. Scroll-triggered reveals pace the reading flow. The pricing toggle removes friction at the decision point. Marquees signal scale without claiming it outright.

What I Practiced

Webflow from first principles

This project was built to demonstrate mastery of the Webflow toolkit — not just the visual output, but the thinking behind structure, performance, and interaction design.

Webflow Interactions — scroll-triggered animations, toggle states, hover effects, all without custom code
Responsive layout system — breakpoints from desktop to mobile without any layout breaks or overflow
Typography hierarchy — headline, subheadline, body, and caption working together across a long-form page
Asset optimisation — .avif and .svg formats throughout for performance without visual compromise
SaaS page architecture — hero → proof → features → pricing → CTA as a deliberate conversion funnel
